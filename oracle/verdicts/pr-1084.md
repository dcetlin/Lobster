## Stage 2 — Quality Review

**Single file changed:** `scripts/claude-persistent.sh`. PR description accounts for all changes. No feature bundling.

**Encoded Orientation check (constraint-3):** This PR changes alert text and adds a classification variable to `write_state`. It does not change behavioral defaults (sleep duration, restart logic, decision-routing rules). Not an Encoded Orientation decision. No logged prior required.

**Detection runs on session-scoped output:** Confirmed. The `current_session_output` variable is produced by the PR #1083 awk expression (session-sentinel scoped). Both the outer trigger grep and the inner classification grep execute on `current_session_output`. Check-4 from the task description passes.

**Pattern reliability:**

Weekly detection: `grep -qi "out of extra usage\|you.re out of"`. The second term `you.re out of` is a partial match fully contained in "you're out of extra usage" — redundant with the first term, but harmless. The `.` in `you.re` is unescaped regex (matches any character), not a literal apostrophe. In context this is inconsequential: no other string of this form would appear in session output. The detection is reliable.

Session detection: `grep -qi "you.ve hit your limit\|hit your limit"`. Same analysis: `you.ve` has unescaped `.`; `"hit your limit"` is a suffix of `"you've hit your limit"` — redundant. Both arms detect session quota. Pattern is reliable in practice.

**Fallback path:** `quota_type` initializes to `"unknown"`. If neither pattern matches (which shouldn't happen given the outer grep already fired on one of these patterns), the else branch provides the original generic alert text verbatim. The fallback is correct and complete.

**What if both match:** The inner if/elif evaluates weekly first. Weekly (`"out of extra usage"`) and session (`"hit your limit"`) are distinct Claude Code quota message phrasings — they do not co-occur in the same quota message. The ordering is irrelevant to correctness; both branches fire only on their respective patterns.

**No sleep/restart logic changed:** Confirmed by diff inspection. `sleep "$sleep_secs"` and the midnight UTC calculation are unchanged.

**One UX asymmetry:** The session quota alert omits the wake time (`$wake_time_et`), while the weekly quota alert includes it. This is arguably correct — session quota resets within hours at an unspecified time, so a concrete timestamp would be misleading. The weekly reset happens at midnight UTC (stated in the PR) and the wake time is accurate. The asymmetry is intentional and defensible.

**Minor technical note:** Unescaped `.` in grep patterns (`you.re`, `you.ve`) matches any character at those positions. Not a correctness risk given the surrounding word context, but technically imprecise. Advisory, not a blocking finding.

**`write_state` change:** The state string now includes `type=session|weekly|unknown`. This is additive. No reader of `write_state` output is broken by additional text in the state value.

**Overall:** The implementation is clean, correctly scoped, and does what it claims. The fallback path exists. Detection runs on the correct (session-scoped) output. No behavioral changes to sleep or restart logic. No Encoded Orientation concern. No feature bundling.
