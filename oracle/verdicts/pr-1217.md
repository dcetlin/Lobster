### Stage 2: Quality Review

#### Finding 1 (NEEDS_CHANGES): Factual error — `restart-mcp.sh` is not a dispatcher restart

The guidance in `restart_services()` states:

> "Manual dispatcher restart IS needed after upgrades that change: [...] Agent bootup files (.claude/sys.*.bootup.md, user config bootup files) — the in-flight session has already loaded these; a new session is required to pick up the changes."
> [...]
> "Safe restart: ~/lobster/scripts/restart-mcp.sh"

This is factually incorrect. `restart-mcp.sh` restarts `lobster-mcp-local` (the MCP server), not the dispatcher (the Claude Code session / `lobster-claude` service). Bootup file changes require a new Claude Code session — a dispatcher restart — which is a different operation entirely.

The CLAUDE.md is explicit: `restart-mcp.sh` exists to safely restart the MCP server without killing the active session. A bootup file change requires a new session. The script cannot deliver a new session; it restarts a server that the existing session connects to.

The guidance as written implies that running `restart-mcp.sh` handles all three change categories (MCP config, bootup files, hook scripts). For MCP config: correct. For bootup files: incorrect — a new Claude session is required, and `restart-mcp.sh` does not provide one. For hook scripts: partially correct (hooks are loaded at session start, so a new session is needed; `restart-mcp.sh` alone does not create one).

The post-upgrade checklist has the same error: "Run: ~/lobster/scripts/restart-mcp.sh" is offered as the action for the case where "this upgrade changed MCP config, bootup files, or hook scripts." For bootup files and hook scripts, this action is insufficient.

**Revision contract for Gap 1:** Separate the guidance by restart type:
- MCP server config changes → `restart-mcp.sh`
- Bootup file or hook script changes → a new Claude Code session is required; the dispatcher does not self-restart for these. The correct mechanism (manual session restart or equivalent) must be named accurately. If the intent is that `restart-mcp.sh` triggers a session reconnect that picks up hooks, verify and document that explicitly; otherwise the guidance is misleading.
