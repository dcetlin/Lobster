## Stage 2 — Quality Review

### Pattern check from learnings.md

The mirror-constant pattern (PRs #696, #800, #967, #970, #974, #980) is a first-pass oracle check per the 2026-04-26 meta learning: "Before reading any PR diff, check: does this PR add a new module-level frozenset, tuple, or named constant whose values match constants in another module?" This check fires on the test file. The test file declares:

```python
NIGHT4_SENTINEL = 5
```

This is a local re-declaration of the production constant `NIGHT4_ROTATION_STATE = 5` under a different name. The test then uses `NIGHT4_SENTINEL` only in fixture calls (e.g., `_write_state(tmp_path, {"current_night": NIGHT4_SENTINEL})`). The production constant is not imported. This is the exact failure mode named across six prior learnings entries: if `NIGHT4_ROTATION_STATE` is ever changed (e.g., the rotation length changes and Night 4 becomes state 6), the test will not catch the regression — the test will still write `current_night: 5` and assert `is_night4_rotation()` returns `True` against a stale value.

The PR comment in the docstring also retains a bare `5` literal: `current_night==5 means Night 4 just finished`. This is acceptable (it is documentation, not a comparison), but it is a separate instance of the value.

### Does it do what it claims?

Yes. The production code is correct: the single use site of the literal is replaced with the constant. The constant's comment is accurate and clarifying. The function docstring already contained the off-by-one explanation — the comment on the constant reinforces it at the definition site.

### Failure modes

The production change introduces no failure modes. The test gap above means the test suite does not guard against a future change to `NIGHT4_ROTATION_STATE` — the tests would silently continue passing with a stale sentinel.

### What patterns does this introduce?

The constant definition follows the StrEnum constants golden pattern's spirit (2026-04-24) — not an enum here, but a named, documented module-level constant replacing a bare literal. That is correct. The test's `NIGHT4_SENTINEL` local declaration propagates the mirror-constant anti-pattern that learnings.md flags as a recurring problem class.

### What this makes harder

If `NIGHT4_ROTATION_STATE` changes value, the test will not detect it. The sole protection is the production import chain — but the test does not import the constant, so it is not in that chain.
