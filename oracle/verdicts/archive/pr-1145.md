### Stage 1: Vision alignment

**Epistemic prior entering review:** This implementation is solving the wrong problem, or solving the right problem in a direction that forecloses better paths.

**What would have to be true for this to be wrong:** The jobs.json enable/disable mechanism is not the real control plane for cron-direct scripts, or there is a cheaper path to the same correctness.

**Finding:** The problem is correctly scoped. Three cron-direct (Type C) scripts — `granola_ingest.py`, `transcription-monitor.py`, and `morning-delivery-flush.py` — ran unconditionally from cron, ignoring their `enabled: false` state in jobs.json. This severed the control-plane contract for `wos start/stop` on these jobs. The gate must live in the script itself (the only execution boundary cron respects for Type C scripts). The problem is real, the fix location is correct, and nothing here forecloses better paths. The duplicate `_is_job_enabled` function definition pattern across all scheduled-tasks scripts is a pre-existing architectural condition, not introduced by this PR — and there is currently no shared utility for it in `src/utils/`.

**Alignment verdict:** Confirmed

Vision anchors:
- `principle-1` (Proactive resilience over reactive recovery): adding the gate before the defect manifests in production is the structural prevention posture.
- `principle-3` (Determinism over judgment for conditionals): the `enabled` field check is a pure boolean gate — code, not LLM instruction.

**Learnings.md effect on Stage 1:** The "migration path mismatch produces silent no-op" pattern (PR #804 re-oracle) was the primary prior I checked at Stage 1. Migration 110 uses `$_m110_jobs_file` correctly derived from `LOBSTER_WORKSPACE`. The path resolves to `~/lobster-workspace/scheduled-jobs/jobs.json` — the same path the Python gate code constructs. No mismatch.

The "feature-bundling" pattern (PR #717 and others) did not fire: the PR modifies exactly four files (three scheduled-task scripts and upgrade.sh). The PR description accounts for all four. No undescribed files.
