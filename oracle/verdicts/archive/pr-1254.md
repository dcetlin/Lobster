### Stage 2: Quality Review

**Does it do what it claims?**

All five tests pass against the real implementation on the PR branch (run via `uv run pytest tests/integration/test_wos_visibility_timeout.py -v` — 5 passed in 2.07s).

**Test design quality:**

- Tests use `tmp_path`-isolated SQLite DBs with real migrations applied via `run_migrations()`. No mocking of registry internals — this is genuine integration coverage.
- Test C and E backdate `claimed_until` via raw SQL after dispatch, which is the correct approach to simulate expired claims without clock manipulation.
- Test D (unexpired claim) correctly validates the negative case: `reset_expired_claims()` must not reset a live claim.
- Test E (multiple expired claims) validates bulk behavior and checks that no UoWs remain in `executing` state after reset — this is the exact failure scenario from the 2026-05-18/19 incident.

**Global state concern:**

`_ISSUE_COUNTER` is module-level global state shared across tests. Since each test uses a fresh DB (via `tmp_path`), and issue numbers are only used for initial UoW seeding, collision is not possible within a single test run. The `issue_offset` parameter in Test E prevents intra-test collision. No risk at present; note for future if tests become parallelized.

**Migration tracking:**

Migration 0025 (`ALTER TABLE uow_registry ADD COLUMN claimed_until TEXT NULL`) correctly stages the schema change that the implementation already depended on. The fact that this was untracked in git is the gap being closed.

**Docstring fix:**

The executor-heartbeat.py docstring update is accurate — it now describes `reset_expired_claims()` and the `claimed_until` model rather than the old `TTL_EXCEEDED_HOURS` recovery path. Matches the actual behavior.

This key exists in the live config but is not in the PR diff — it was added in a prior session, not in this PR. The registry uses `estimated_runtime * 2` with a 3600s fallback, not this config key. This is a pre-existing orphaned config key outside this PR's scope. Not a gap in this PR; flagged as a pre-existing observation.
