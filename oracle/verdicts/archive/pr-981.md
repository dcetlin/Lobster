## Stage 2: Quality review

### Does it do what it claims?

Yes. The four branches (pipeline paused, all-orphan auto-retry, mixed, all-non-orphan) are implemented and tested. The dispatch table entry and `route_wos_message` fast-path are correctly wired. ### Key decisions examined

**`_SURFACE_ORPHAN_RETURN_REASONS` vs. `ORPHAN_REASONS` in steward.py:** Not a mirror-constant violation. `ORPHAN_REASONS` contains 3 values; `_SURFACE_ORPHAN_RETURN_REASONS` contains 5, adding the two heartbeat-classified orphan return_reasons. The difference is deliberate: `metadata.causes` carries raw return_reason strings that can include heartbeat-classified values not in `ORPHAN_REASONS`. The constant and comment are accurate.

**Oracle finding Round 1 â€” mixed-branch `orphan_note` text inaccuracy:** The original implementation said orphan UoWs in the mixed branch "were auto-retried" when no retry subagent was actually spawned. This was inaccurate. The dispatch architecture allows one action per message; the mixed branch must surface non-orphans via `send_reply`, so orphan UoWs cannot be auto-retried in the same result. **Fixed in commit `e97a8b08`:** the `orphan_note` text now lists each orphan UoW with its `/decide retry` command so Dan can act explicitly, and adds an inline comment explaining the architectural constraint.

### Remaining observations (non-blocking)

1. **All-orphan branch does not notify Dan:** The T1-A spec says "send Dan a summary notification" for all-orphan auto-retry. The current implementation spawns the steward heartbeat silently (consistent with `handle_wos_escalate` branches 1/2). A brief "Batch-reset N orphan UoWs from kill wave" notification would require either a dispatcher-level side-effect or a follow-on PR. Not a blocker â€” behavior is safe and consistent with precedent.

2. **Deterministic `task_id` on count only:** `f"surface-batch-retry-{len(orphan_uow_ids)}uow"` means two same-size kill waves produce the same `task_id`. Rare event; session-start hook will reject the duplicate with a `send_reply` alert via the exception catch in `route_wos_message`. Acceptable.

3. **Mixed branch does not spawn an actual retry subagent:** Architectural constraint acknowledged in code comment. Dan receives explicit `/decide retry` guidance for orphan UoWs. Correct behavior.
