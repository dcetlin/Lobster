**Security checks:**

1. **Token read from env, not hardcoded:** The bash reads `LOBSTER_REVIEW_TOKEN` from `~/lobster/config/config.env` via `grep -E '^LOBSTER_REVIEW_TOKEN=' ... | cut -d= -f2-`. Runtime read, not compile-time constant. Correct.

2. **Token leakage in logs or error messages:** The `||` warning branch echoes a fixed string: `"Warning: LOBSTER_REVIEW_TOKEN set but gh api review failed (see output above). Option B soft gate still applies."` — this string does not contain the token value. The `tee -a` pipes only this fixed string to the verdict file. The `2>&1` captures gh api stderr but this goes to stdout (visible in the oracle session log), not to the verdict file. No token leakage path exists in the verdict file or normal log outputs. Under normal `gh` CLI operation (without `GH_TRACE=1`), the token does not appear in gh output.

3. **Fallback when token absent:** `if [ -n "$REVIEW_TOKEN" ]; then ... fi` — when `REVIEW_TOKEN` is empty or unset, the entire block is skipped. Silent skip. The Option B gate is unchanged.

4. **Fallback when gh api call fails:** `|| echo "Warning: ..." | tee -a verdict_file` — the warning is appended to the verdict file and the oracle run continues. The `VERDICT: APPROVED` line (already written at that point) is unaffected. Option B dispatch proceeds normally.

5. **Shell pipe precedence:** `gh api ... || echo "Warning..." | tee -a file` — due to pipe precedence, this parses as `gh api ... || (echo "Warning..." | tee -a file)`. The tee applies only to the warning echo, not to gh api output. Correct — only the warning string reaches the verdict file.

6. **Template variable substitution:** The bash block uses `{owner}/{repo}` and `{pr_number}` as template placeholders. The instruction text explicitly tells the oracle to "Extract owner/repo and pr_number from context available at runtime." This is consistent with the oracle instruction's existing use of `{number}` in verdict file paths. The LLM executing this block is expected to substitute from context — same as all other `{...}` placeholders in lobster-oracle.md.
