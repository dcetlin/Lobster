The vision emphasizes Lobster as a cognitive extension that reduces friction and screen time without substituting for judgment (constraint-4, principle-2). The question entering Stage 1: is this fixing the right layer? The session-note-polish agent works by reading accumulated snapshot blocks. If snapshot blocks contain sufficient information to correlate a thread-level finding with a task completion, then the fix is at the correct layer. If the data is absent or uncorrelated, the instruction is a dead letter — the agent can't find what it can't see.

The residual risk is for vague findings with no task anchor — these can't be superseded by the instruction regardless of phrasing. The instruction handles this correctly by scoping to "when a finding references work that was completed" — it only fires on correlatable cases.

**Alignment verdict:** Confirmed. The fix serves constraint-4 at the correct layer, assuming completion signals are present in snapshot data.

## Stage 2 — Quality Review

### Diff scope

Single sentence added to the Open Threads instruction in `session-note-polish.md`. No other changes.

### Does it do what it claims?

The instruction tells the agent to check whether an Open Threads finding references work completed earlier in the session, and if so, prefix the entry with `[superseded — closed by task-id X]`. The agent has access to snapshot blocks, which contain task completion data. The instruction is operationally coherent: the data the agent needs is in scope, the trigger condition is named, the output format is specified, and a concrete example is provided.

### Over-constraint check

No. The instruction is conditional ("when a finding references work that was completed"), not unconditional. Non-superseded findings are unaffected. The existing instruction ("keep only what is genuinely unresolved") is extended, not overridden.

### Contradiction check

No contradiction. The existing instruction establishes the principle (keep only genuinely unresolved items); the new instruction adds a specific handling pattern for the edge case where items appear unresolved but were resolved within the session. These are consistent.

### Failure modes

The instruction can only fire when the agent can correlate a finding to a completion signal. For vague findings without a task anchor, the agent has no basis for marking superseded — and the instruction scopes itself to "references work that was completed," so it correctly does not fire on ambiguous cases. The residual failure mode is under-detection (some superseded items not caught) rather than over-detection (correct items incorrectly suppressed). Under-detection is the safer failure mode for this instruction type.

It does not change system-level routing defaults, dispatching rules, agent identity, or decision-making gates. Constraint-3 (Encoded Orientation) does not require a logged prior for a behavioral correctness fix to a single agent's output format. Not flagged.
