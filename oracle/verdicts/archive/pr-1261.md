## Stage 2: Quality Review

### Does it do what it claims?

Yes. `generate_v2_prescription()` produces a `PrescriptionV2` dataclass conforming to the 7-section schema. The function reads UoW state, diagnosis, vision_ref, Dan's register, and issue_info. It writes one audit entry per call. It falls back to deterministic generation if LLM fails.

### Encoded Orientation check (constraint-3)

This PR adds a new Steward output path (`prescription_v2` audit event, new dataclass). This constitutes an Encoded Orientation change to Steward behavior. Checking:
- (a) Prior logged decision: spec issue #573 and implementation issue #574 are both logged. The prescription format spec doc is already committed and reviewed.
- (b) Vision.yaml anchor: vision_ref is baked into the generation path; `generate_v2_prescription` calls `resolve_vision_route()` when `uow.vision_ref` is present, and the resulting orientation string is passed to the LLM. The audit_metadata carries the uow_id and schema_version, enabling traceability.
- Verdict: constraint-3 satisfied.

### Robustness of fallback chain

`_llm_prescribe_v2` falls back to `_build_v2_prescription_deterministic` on: subprocess error, non-zero returncode, empty stdout, JSON parse failure, missing required sections. The chain is exhaustive for known failure modes. `_process_uow` wraps the entire v2 generation in `except Exception` so v1 WorkflowArtifact write is never gated on v2 success.

### PrescriptionV2 dataclass

Frozen, slotted — correct choice for an immutable output artifact. `to_json()` is a clean serialization with no circular reference risk. The 7 fields map 1:1 to the spec sections.
