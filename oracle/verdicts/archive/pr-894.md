## What this forecloses

The `needs-human-review` status is now a named anchor in the UoW lifecycle. Future changes to escalation behavior (per-register retry thresholds, different escalation channels, auto-close after N days without response) must treat `needs-human-review` as an existing status rather than introducing a new one. The non-terminal designation (`is_terminal()` returns False) preserves the correction path — a human can transition out of this status.

## Patterns introduced

The decision-file-as-Encoded-Orientation-prior pattern (a markdown document in `oracle/verdicts/` that explicitly authorizes an autonomous behavior class) is now established as the mechanism for satisfying constraint-3 without a vision.yaml `open_decisions` entry. This is a lighter-weight path than a vision.yaml edit for operational behavioral extensions — appropriate when the anchor is already present in vision.yaml and the decision is a concrete implementation of a named principle.
