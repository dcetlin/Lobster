## Stage 1: Vision Alignment

**Finding (written before reading implementation):**

The PR description frames this as noise reduction: garden-caretaker's every-15-minute cadence generates `event='skipped'` audit entries for every in-progress UoW, producing 675k+ zero-signal rows. The claim is that `UpsertSkipped` already encodes the outcome structurally to callers, so the audit write adds nothing.

The theory-of-change question for Stage 1 is: does this work serve the WOS substrate, or does it undermine observability that the substrate depends on?

The vision's active_project.phase_intent names the Registry as the structural anchor for work-state visibility. Vision principle-1 ("Proactive resilience over reactive recovery") and principle-4 ("Integration rate before new feature rate") both favor keeping existing instrumentation paths intact unless there is positive evidence they are producing only noise. The risk entering Stage 1 is: the `skipped` audit entries may be low-signal in their current volume, but removing them could silence a diagnostic path that future callers or monitoring would use.

What would have to be true for this to be the wrong path: (a) some monitoring or alerting actually counts `event='skipped'` rows, or (b) the audit trail is used forensically to reconstruct "how many times was this UoW skipped before it was re-created" — and that answer has value.

The key question is answerable by grep, not by design judgment. Stage 2 will decide the verdict.

**Alignment verdict: Confirmed** — noise reduction in the Registry's audit trail directly serves the "integration rate before new feature rate" principle by reducing DB churn that makes the audit log harder to query for signals that matter. The change is in the right direction for the phase.

This PR does not change behavioral defaults, system constraints, agent identity, or decision-making rules. It removes a side effect from guard-clause early returns. Not an Encoded Orientation decision. Constraint-3 does not apply.

The 675k+ `skipped` entries continue to accumulate. No alert fires, no routing decision changes — the noise simply persists. The cost is DB size and query performance degradation for legitimate audit queries.

None identified.
