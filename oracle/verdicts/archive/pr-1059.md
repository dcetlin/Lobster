## PR #1059 — feat: Phase 2d backlog alerts (toxicity + starvation governor)

### Stage 1: Right problem?

Yes. The WOS pipeline needs visibility into two failure modes:
- Toxicity: executor under-capacity relative to cultivator output (queue growing monotonically)
- Starvation: cultivator/germinator throughput death (queue empty for N cycles — not idle rest)

These are distinct asymmetric failure modes that require different remediation. Detecting them early via the steward heartbeat is the correct architectural choice. Running Phase 2d *before* the `execution_enabled` gate ensures monitoring is active even when WOS is paused — this is correct: a paused system can still be accumulating or starving.
