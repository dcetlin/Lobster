## Prior Gap Tracking

Three blocking gaps from Round 1. Status of each:

**Gap 1 (dead constant `JUICE_MIN_COMPLETION_RATE`):** Addressed.
`_score_completion_rate` now contains `if rate < JUICE_MIN_COMPLETION_RATE: return 0.0` before returning the normalized ratio. The behavioral test `test_completion_rate_below_minimum_returns_zero` asserts the gate consequence (1 done out of 4 cycles = 25% → returns 0.0) rather than the constant value alone. The constant-value test `test_minimum_completion_rate_is_30_percent` remains alongside it — this is acceptable; the revision contract required adding a behavioral assertion, not removing the constant test.

**Gap 2 (private `_connect()` access from `compute_juice`):** Addressed.
`Registry.fetch_audit_entries(uow_id)` is added as a public method in registry.py. It wraps `_connect()` internally (which is correct — the Registry owns its connection management). `compute_juice()` now calls `registry.fetch_audit_entries(uow_id)` at the public boundary. Tests updated to mock `fetch_audit_entries` instead of `_connect`.

**Gap 3 (no logged Encoded Orientation decision):** Addressed per the revision contract's stated terms.
`oracle/decisions.md` is created with ADR-001 recording Dan's explicit approval ("good lets implement" — 2026-04-24), the authorizing design thread (#886, #888, #889), the dispatch constraint, and the starvation sequencing follow-on noted by the oracle. The revision contract asked for "a logged decision record" — that record exists.

Advisory (non-blocking): ADR-001 does not explicitly cite a vision.yaml field path as the anchor. Constraint-3 requires "a traceable vision.yaml anchor."
