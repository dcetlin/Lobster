## Stage 2: Quality Review

### What the diff does

Single-line change in `src/orchestration/dispatcher_handlers.py` line 43:
- Before: `from utils.timezone import format_iso_for_user as _format_iso_for_user, get_owner_tz_name as _get_owner_tz_name`
- After: `from src.utils.timezone import format_iso_for_user as _format_iso_for_user, get_owner_tz_name as _get_owner_tz_name`

### Does it do what it claims?

Yes. The root cause is verifiable from the runtime context:

- `steward-heartbeat.py` runs via cron as `cd ~/lobster && uv run scheduled-tasks/steward-heartbeat.py`
- The script adds `~/lobster` (repo root, `_REPO_ROOT`) to `sys.path`
- When steward-heartbeat imports `is_execution_enabled` from `src.orchestration.dispatcher_handlers`, Python executes the dispatcher_handlers module
- At module load, `from utils.timezone import ...` fails because `utils` is not a top-level package at `~/lobster` — it lives at `~/lobster/src/utils/`
- The fix `from src.utils.timezone import ...` resolves correctly because `src` is a top-level package at the repo root (verified: `uv run python3 -c "from src.utils.timezone import format_iso_for_user, get_owner_tz_name; print('Import OK')"` passes)

**Why unit tests didn't catch this before:** The test conftest.py inserts `SRC_DIR` (`~/lobster/src`) into `sys.path`, making `from utils.timezone` resolve in test context but not in the cron-invoked runtime context. This is a known class of test-environment/production-environment sys.path divergence.

### What I found that the engineer didn't flag

**The out-of-scope bare imports are a partial risk.** The PR correctly identifies other files with bare `from utils.X` imports and defers them. The engineer states those "may have their own sys.path setup." This is partially accurate:

- `src/mcp/inbox_server.py`, `src/bot/lobster_bot.py`, `src/bot/sms_router.py`, `src/channels/outbox.py`, `src/mcp/message_store.py`, `src/mcp/reliability.py` — these all contain bare `from utils.X` imports but run in the context of the MCP server, which may insert `src/` into sys.path via a different mechanism
- `src/delivery/circadian.py` — has `from utils.timezone import get_owner_zoneinfo` with no sys.path manipulation visible in the file itself

The deferred files represent latent risk of the same class if their runtime invocation context changes. However, the PR's scope is explicitly limited to the one import causing the confirmed live crash, and the deferred files were not part of the crash traceback. This is defensible triage. The risk is noted but is not a blocker for this PR — it merits a follow-up issue.
