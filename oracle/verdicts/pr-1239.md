### Stage 2 — Quality Review

#### Gap 1 (blocking): Four commands in `PRE_HANDLER_COMMANDS` are silently unreachable as slash commands

`pre_handler.py` defines handlers and registers `/subagents`, `/jobs`, `/wos`, `/restart` in `PRE_HANDLER_COMMANDS`. It also imports and exports `handle_help_command`. However, `lobster_bot.py` only registered three `CommandHandler` entries.

The `try_handle()` path is called from `handle_message()`, which is registered under `MessageHandler(filters.TEXT & ~filters.COMMAND, ...)`. The `~filters.COMMAND` filter means Telegram slash commands never reach `handle_message()` — and therefore never reach `try_handle()`.

**Result:** `/subagents`, `/jobs`, `/wos`, `/restart`, and `/help` were in `PRE_HANDLER_COMMANDS` with implementations, but were silently dropped by the bot layer when sent as slash commands.

#### Gap 2 (minor): Private function import from `wos_dashboard`

`_handle_wos()` imports `_get_bisque_relay_base_url` from `orchestration.wos_dashboard`. The leading underscore marks this as module-private. The function was already wrapped in a `try/except` with a graceful fallback.
