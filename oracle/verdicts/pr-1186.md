## Stage 2: Quality Review

### Does it do what it claims?

**1. Label self-validation (Step 1b):** The diff fixes the label name from `wos-uow` to `wos:uou` and adds a `gh label list` pipe to grep before trusting a zero result. The logic is correct. One note: the instruction says "Apply the same validation logic to any other label-based queries in this step (e.g., `wos:executing`)." This is advisory, not structural — the sweep agent must remember to apply it. This is acceptable for an instruction document but could become the target of a future improvement (structural pre-validation of all labels used in the step). Not a blocker.

**2. execution_enabled gate (Step 1b):** The new step 4 reads `wos-config.json` and gates stall prescriptions on `execution_enabled`. The logic flow is clear: if gate fires, record the reason and skip steps 4a and 5. The renaming of the original step 4 to step 4a is structurally clean. The instruction correctly handles the `file exists + field false` case. One edge: the instruction does not specify what to do if the file does not exist — but the existing behavior (treat as not-paused, proceed to step 4a) is the correct default per the CLAUDE.md description of wos-config.json ("File absent — treated as false (safe default)"). The instruction does not contradict this — it says "If the file exists and `execution_enabled` is `false`." Correct.

**3. Contradiction Matrix dead-entry detection (Step 1g, Night 2 only):** The step lists seven specific paths to pre-validate before running matrix checks. The paths listed match the files the Contradiction Matrix logically references. The instruction handles missing files correctly (surface as entropy item, count consecutive-absent sweeps). One structural observation: the step is placed after Step 1e and Step 1f in the document (based on prior numbering) but scoped to Night 2 only, same as the matrix checks it validates. This is correctly sequenced — Step 1g fires before Step 2, as the diff states.

   The instruction "Check how many consecutive sweeps this file has been missing (look at recent sweep files in `~/lobster-workspace/hygiene/`)" is workable but requires the sweep to read its own output history. This is an existing pattern in the sweep (Night 7 synthesis reads prior nights). Not a new pattern, not a new risk.

**4. OODA Act-layer gap (Step 4 Escalation list):** The standing instruction checks two compound conditions (execution paused + old escalation items with no action). The priority item text is concrete: it names the dependency chain, requires specific issue numbers and ages, and is placed at the top of the Escalation List section — not buried. The use of `--label "needs-decision"` as the proxy for escalation items is approximate (not all escalation items have this label, and not all needs-decision items are Act-layer gaps), but the instruction says "or query the escalation list items directly by issue number" — giving the agent flexibility. This is appropriate for an instruction document that must handle partial information.

### Encoded Orientation check (OODA constraint-3):

None of the four changes encodes a new behavioral default for agents outside the sweep context. The changes are all within the sweep instruction document, governing only how the sweep agent behaves when it runs. The sweep agent itself is a scheduled job that produces output for Dan — it does not make routing decisions or take autonomous actions beyond its existing autonomy calibration section. No constraint-3 trigger.

### Failure modes:

- **Step 1b label validation false negative:** The grep `grep -q '^wos:uou$'` uses an anchored match, which is correct. If the label name changes in the future, the hardcoded string will lag. Acceptable — instruction documents require periodic maintenance.

- **Step 4 execution_enabled gate omission on step 5:** The diff correctly notes "and execution_enabled is not false" in the amended step 5 instruction. The gate is applied at both the step-4a and step-5 level.

- **Step 1g path list staleness:** The seven paths listed are specific. If the system restructures and a path moves, Step 1g will surface false dead-entry signals. This is acceptable — the signal will be correct (path does not exist), and the sweep agent is equipped to investigate. The risk is false-positive entropy items, which is lower-severity than false-negative silence (the status quo before this PR).

### What this makes easier or harder:

- **Easier:** Diagnosing sweep-level false-zero results. The execution-paused state is now explicitly distinguished from executor starvation. The Contradiction Matrix is self-validating on Night 2. The OODA Act-layer gap is machine-surfaced rather than human-noticed.

- **Harder:** Nothing. The additions are instruction-document prose; they do not introduce code patterns that propagate.
