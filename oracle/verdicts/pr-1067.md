## PR #1067 — fix(tests): add mirror-constant lint, fix 25 violations

### Stage 1: Right problem?

Yes. The oracle has caught the same test anti-pattern four times (PRs #696, #800, #967, #970): test files re-declare production constants as module-level literals instead of importing them. When the production constant changes, the test constant silently drifts — the assertion passes against a stale value and the test gives false confidence.

This PR:
1. Adds structural enforcement (AST-based lint test) that detects the violation
2. Fixes all 25 existing violations across 13 test files

This is the correct level of intervention — structural enforcement closes the feedback loop permanently.

### Stage 2: Well made?

**Lint test (`tests/test_no_mirror_constants.py`):**
- AST-based: walks `ast.Assign` and `ast.AnnAssign` at module level — no false positives from local variables or function-scoped constants
- Builds a production constant index from `src/` and `scheduled-tasks/` — correct scope
- `MIRROR_CONSTANT_ALLOWLIST` for genuinely coincidental cases (`LOBSTER_JID`, `ADMIN_CHAT_ID`)
- Pure functions: `_collect_module_level_constants`, `_build_production_index`, `_scan_for_mirror_constants` — correct primitives

**25 violations fixed:** Each fixed file now either imports the constant from the production module or accesses it via the already-loaded module attribute (e.g. `qm.STARVATION_CONSECUTIVE_READINGS`).
