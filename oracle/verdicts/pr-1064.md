## PR #1064 ‚Äî feat: reconciler uses workstream/status.md as liveness signal

### Stage 1: Right problem?

Yes. Long-running dispatches update `workstream/{task_id}/status.md` every ~5 minutes per the Long-Running Dispatch Preamble in CLAUDE.md. A stale status.md is a reliable earlier stall signal (10 min) than the existing output_file mtime gate (15 min). This is the correct place to add this check ‚Äî the reconciler already has an mtime gate for output files; adding the workstream gate as an OR-condition is a minimal, targeted extension.

### Stage 2: Well made?

**`get_workstream_status_mtime()` in session_store.py:**
- Pure function: reads filesystem metadata only, no side effects
- Returns `None` for empty `task_id`, missing directory, or missing `status.md` ‚Äî conservative fallback is correct (better to not kill than to false-positive)
- `WORKSTREAM_BASE` is a module-level constant, monkeypatch-testable

**Reconciler integration in inbox_server.py:**
- `ws_mtime = get_workstream_status_mtime(task_id) if task_id else None` ‚Äî correct guard
- `effective_stale = ws_is_stale or file_is_stale` ‚Äî OR logic is right; either signal collapsing to short threshold is correct behavior
- Stale note in log message distinguishes workstream vs output_file gate clearly
- `result_summary` correctly appends workstream staleness detail when ws gate fired
- Sessions without workstream dirs are fully unaffected ‚Äî no new DB columns, no config changes
