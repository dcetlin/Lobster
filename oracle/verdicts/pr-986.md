## Stage 1 — Vision alignment (formed before reading implementation)

The PR's stated problem: `claude -p` receives the prompt as a positional argument; when the prompt begins with `---` (YAML front matter), the CLI parser treats it as an unknown option and exits 1, causing every WOS UoW dispatch to fail.

What would have to be true for this to be the wrong path: `-p` would have to be a value-taking option, in which case the prompt would already be consumed as its value and the `---` parsing issue would not exist. The PR description explicitly states the premise ("boolean flag, not value-taking") and provides manual test confirmation. The adversarial prior is not confirmed by available evidence.

The fix is directly in service of the active project phase intent: WOS substrate stability. A PR that stops all WOS UoW dispatch from failing on YAML-prefixed prompts is unambiguously aligned with `active_project.phase_intent`. No foreclosure of paths.

**Alignment verdict: Confirmed**
